GitHub 是个宝藏，让我们来探索它。当我们打开一个仓库的设置页面时，我们会在它的侧边栏那面发现一个 webhook 的选项，点击它。


其中

  1. payload url 就是当 GitHub 的 webhook 机制触发时它要传输的数据的地址。（注意，这里先不写飞书的 webhook 地址）;
  2. content type 是我们传递数据的格式，是 json 还是 xml , 我们这里选 json 格式。 application/json
  3. secret 是密钥
  4. 就是选择触发 webhook 的事件。

# 编写中间件过滤 GitHub 请求数据

其实当我们去看 GitHub 传输的数据，我们就会发现它的数据格式和飞书机器人接受的数据类型是不同的！！！
那么我们就需要做一个请求拦截、过滤、转发的中间件。
使用我们较为熟悉的 node 去写，博主这里是使用了 express。

代码很简单，因为我们的功能就很简单。
但是这里有一个问题就是，我们的中间件部署后才能使用，本地部署的话，GitHub 的 webhook 地址是不允许我们使用本地的地址的。所以我们就需要把我们的中间件项目部署到我们的服务器上。
这里推荐使用 forever + nginx 进行线程持久化部署。也可以用 docker, 达成目的就行。
https://www.qetool.com/scripts/view/25655.html

然后我们把我们部署后的项目地址填到 GitHub 的 webhook 的 payload url 中。我们就可以进行测试啦。
