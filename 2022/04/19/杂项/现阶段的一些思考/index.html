# 正文

深夜一点更文，对自身技术之路的一些思考，我并不认为这是一种 emo。我只是相对自己进行一个梳理。

之前我的好友 安全-朱师傅 写了他对安全这条道路的一些思考
也对我有一些触动。

回想一下一年半前 (其实不到), 开始学前端，以来自己不管技术方面还是其他方面都学了很多。
细数一下，自一开始的 Html , Css , JavaScript 的前端三剑客，然后写了当时的一个科协作业。

比较有趣的是，我魔改了当时的作业需求，在默认需求之外，添加了狂暴模式 (–>), 利用了学长写的后端接口的一个规律性，暴力的把当时的所有学前端的人的私人数据都请求了下来 (定时器轮询，30ms 一次) 当时还想学长会不会知道，在群里骂娘 (还有些期待)。

之后就去学了前端的框架 Vue （以及它的生态）, UI 框架 Element UI 并且也使用 Vue 写了一个简单的社团报名系统 (伪，因为功能特别少).
然后就开心进行了前端第一次考核，然后最终考核这个样子，中间学了一点 Node.js , less , sass (其实和 less 基本一样，不过这个好像强点), 也磕磕绊绊的进了中心 (当时有点浪，面试就是奔着唠嗑去的 (不知名的学长背锅))。
慢慢的走过来，学了 Node.js 框架 express , 数据库 mongoDB , 学了原生的 微信小程序 ， uniapp , 还有一些小程序 UI 框架的使用。后面又去学习了 Webpack , Vue3 , TypeScript 。
然后为了部署项目又学了
Linux , Nginx , docker 还有服务器的一些操作。
然后看了点 计算机网络 ，以及 js 的一些可能算核心的原理，Vue 的一些原理，用 js 去实现数据结构。
为了写一个项目学了 echarts (里面的文档基本熟悉)

这里还有一些比较细碎的知识点，就不说了

然后为了熟悉技术，写了一个基于 Vue 的商城和一个后台管理，一个基于微信小程序的商城，一个基于 uniapp 的商城 (就是前面的翻版), 一个基于 node + express + mongoDB 的项目，一个基于 uniapp 的社团报名小程序，一个基于 Vue3 + Ts 的后台管理，一个基于 vue2 + echarts 的可视化网页 (主要是美观). 还有一些别的小项目就不说了，其实还写过很多的静态页面 (响应式的和不响应的)。
现在回头看看，确实也学的不少 (可能不多，大佬勿喷)。现在又急赶着想去实习 (被风气影响了？)。其实我感觉是受到了一些出去实习的学长的影响，比较羡慕吧。

但是返回头想一想，我们去实习的目的是什么？是学技术，混一个实习经历。 但是我真的掌握好我学的这些东西了吗？其实不然，因为各种各样的原因，学的杂又快，并不能说是熟练，只能说是熟悉或者是了解基本操作。一句话总结：现在的节奏太快，之前也没有一个高效的学习方法。导致一个知识点不断的反复。所以我真的该去人云亦云去实习吗，大众效应？
我的目标和想法是什么？什么才是我真真该做的？滚去梳理自己的知识体系。

# 后面的计划

利用写项目，写博客总结，来梳理之前的知识。
