#### 前言

我们经常使用着 javaScript,但是却有时候并不知道它是为什么去运行,是基于什么机制去执行。只是单纯的会用就行。那么我们了解的 js 是什么呢？当我们说起它, 我们会想起什么呢？ 动态弱类型语言, 解释性语言, 单线程, 非阻塞。但是它为什么是单线程, 非阻塞。内部是如何实现？还是会有一些疑问。

#### 正文

js 这门语言是专门设计来与浏览器进行交互的脚本语言, 那么作用决定性质, 所以他就有了单线程与非阻塞的特性。单线程是为了在 js 操作 DOM 的时候不会由于多个操作操作同一个 DOM 元素而导致(避免样式错乱), 同一个时间只能做一件事。非阻塞是为了让用户在进行一个高耗时的操作后, 不至于页面卡死。

那么我们下面来重点说一下非阻塞

#### 非阻塞

非阻塞其实理解起来就是把一个"我们需要它异步"操作的事件挂起来, 让它去执行, 等有结果再去处理它, 那么把它挂在哪呢？挂起来后什么时候去处理？如何处理？这是个问题。

##### 执行栈与事件(任务)队列

- 执行栈(execution context stack)是程序代码执行的地方, js 程序会按照从上到下的顺序去执行代码, 把需要执行的代码依次入栈, 执行完再出栈。
- 任务队列(task queue)在执行栈外, 当异步任务返回结果就会向 task queue 中放一个事件。
- webApi, 异步事件挂载到这, 等它返回结果。

那么他们具体是怎么处理呢？
:::info
其实就想上面那个图一样, js 会先处理执行栈中的代码，遇见需要异步处理的事件就将它挂到 webApi 上,等异步事件返回结果就将它加到任务队列中, 然后等执行栈中的事件处理完空闲的时候就去处理任务队列中的事件。并且不断去重复这个过程, 我们称之为事件循环。
:::
那么我们就需要知道另外一个知识点: 宏任务和微任务

###### 宏任务和微任务

上面的任务队列中其实又分为宏任务(macro task)和微任务(micro task), 两者又有不同的队列。
那么我们为什么要有两种任务队列呢？
:::info
页面渲染事件，各种 IO 的完成事件等随时被添加到任务队列中，一直会保持先进先出的原则执行，我们不能准确地控制这些事件被添加到任务队列中的位置。但是这个时候突然有高优先级的任务需要尽快执行，那么一种类型的任务就不合适了，所以引入了微任务队列。 <a href="https://segmentfault.com/a/1190000022805523">来源</a>
:::

- macrotasks: 包含生成 dom 对象、解析 HTML、执行主线程 js 代码、更改当前 URL 还有其他的一些事件如页面加载、输入、网络事件和定时器事件。
- microtasks: primose 事件

那么 js 代码如何执行呢？
先在执行栈中执行, 然后有异步任务就放到 webApi 中, 等它返回结果就放到任务队列中, 并且分为宏任务和微任务。当执行栈中空闲的时候, 就进行事件循环。事件循环的模式为: 先看一下宏任务中是否有需要执行的任务, 如果有就取一件出来执行, 然后去执行由它产生的微任务。如何没有就去看微任务。然后去循环。
