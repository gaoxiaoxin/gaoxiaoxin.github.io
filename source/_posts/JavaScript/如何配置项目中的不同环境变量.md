### 区分不同环境实现环境变量的切换

在开发中，我们经常需要根据不同的环境设置不同的环境变量，常见的环境有三种:

1. 开发环境: devlopment
   对应于 vue_cli 项目中的 npm run serve
2. 生产环境: production
   对应于 vue_cli 项目中的 npm run build
3. 测试环境: test
   对应于 vue_cli 项目中的 npm run test

那我们如何在项目中去根据环境的不同去更改 BASE_URL 呢？
常见的方式是:

1. 手动修改不同的变量
2. 根据 process.env.NODE_ENV 的值进行区分
3. 编写不同的环境变量配置文件

#### 手动修改不同的变量

没错，你没想错，就是单纯的手动，在需要你!!人工(智能)!!{.bulr}去修改不同环境下的变量

```javaScript 示例
// devlopment
let BASE_URL = "http://xiaoxx.org/dev"
// production
let BASE_URL = "http://xiaoxx.org/prod"
// test
let BASE_URL = "http://xiaoxx.org/test"
```

需要哪个，就自己去换。。。  
:::danger
不推荐
:::

#### 根据 process.env.NODE_ENV 的值进行区分

```js 示例
let BASE_URL = "";
const TIME_OUT = 10000;

if (process.env.NODE_ENV === "development") {
  BASE_URL = "http://xiaoxx.org/dev";
} else if (process.env.NODE_ENV === "production") {
  BASE_URL = "http://xiaoxx.org/prod";
} else {
  BASE_URL = "http://xiaoxx.org/test";
}
export { BASE_URL, TIME_OUT };
```

然后再该使用的地方使用就可
::: success
推荐
:::

#### 编写不同的环境变量配置文件

1. vue 官方配置
   https://cli.vuejs.org/zh/guide/mode-and-env.html#%E6%A8%A1%E5%BC%8F

2. ![](https://tva1.sinaimg.cn/large/006xVoHRly8gwv9xz3rhej307602mq2u.jpg)
   如图所示：自定义三个.env 文件

自定义文件后，需要重新启动项目
三个文件的配置

```js development
VUE_APP_BASE_URL = "http://xiaoxx.org/dev";
VUE_APP_BASE_NAME = "xiaoxx";
```

```js production
VUE_APP_BASE_URL = "http://xiaoxx.org/prod";
VUE_APP_BASE_NAME = "xiaoxxprod";
```

```js test
VUE_APP_BASE_URL = "http://xiaoxx.org/test";
VUE_APP_BASE_NAME = "xiaoxxtest";
```
