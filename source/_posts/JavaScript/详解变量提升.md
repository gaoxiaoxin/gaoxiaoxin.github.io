#### 变量提升

对于很多学习 js 的小伙伴来说,var 的变量提升和函数优先这些概念应该都不会陌生, 那么现在让我们去看一下

由于 js 是一个解释型语言, 所以 js 的代码会一行一行的执行, 那么当我们在一个变量进行声明之前对它进行调用， 是不是就会报错显示这个变量是 not defined 呢？

```js demo
console.log(a); //  undefined
var a = 1;
```

```js demo
foo(); // 1
function foo() {
  console.log(1);
}
```

其实我们会输出 a 为 undefined,而不是我们想的那个 not defined ; 那么是为什么呢？其实这个是因为++Javascript++++引擎++会在代码执行前对代码进行一个++预解析++, 先获取所有被声明的变量，然后一行一行地运行。这样的话就会造成 var 变量和函数声明可以在代码未执行到它的时候进行使用，这就是我们现在讲的重点 ==变量提升==

那么什么是提升(Hoisting)呢？
:::primary
++变量提升(Hoisting)++ 被认为是, Javascript 中++执行上下文++(特别是创建和执行阶段)工作方式的一种认识。
:::
简单讲就是 js 引擎会在++预编译期间++将变量声明和函数声明提升到对应作用域的最顶层
下面来看一段代码

```javaScript demo
console.log(foo)
function foo() {
  console.log(2);
}
var foo = 1
console.log(foo)
```

上面的话第一个打印输出 foo 为 foo 函数, 而第二个打印输出 foo 就显示 foo 变为了 1
然后再看一个，大家可以想一下，这个会输出什么

```js demo
console.log(a);
var a = 1;
function foo() {
  console.log(a);
  var a = 2;
}
foo();
```

会输出两个 undefined, 其实代码进行一个转换后是这个样子

```js demo
var a;
console.log(a);
a = 1;
function foo() {
  var a;
  console.log(a);
  a = 2;
}
foo();
```

通过这两个例子大家应该对变量提升有了一个更好的认识, 但是在第一个例子中, 我们发现当函数声明和变量声明同时存在的情况下, 在++两者声明之前++foo 是函数, 那么这个就是我们老生常谈的 ++函数优先++
:在函数名和变量名冲突的时候，js 引擎预编译期间会优先进行函数的声明。官方一点讲就是，js 在预解析的时候++首先会处理函数声明，其次会处理变量声明，如果如果变量名称跟已经声明的形式参数或函数相同，则变量声明不会干扰已经存在的这类属性++。
:::info
函数优先(函数声明优先) : 注意我们这里说的函数优先是指的是函数声明优先,并不是指函数表达式优先
:::

```js demo
console.log(foo); // undefined
var foo = function () {
  console.log(2);
};
```

现在的话我们是讲完了 var , 但是还有人说 js 现在不是还可以用 let , const 对变量进行声明吗,他们还不像 var 这样, 他们只带块级作用域。我们现在来看,上个代码看一下:

```js demo
console.log(a);
let a = 2;
```

```js demo
console.log(a);
const a = 2;
```

当我们运行这个代码，编译器就会给我们报一个错误 : ++Cannot access 'a' before initialization++, 翻译为汉语就是: 在 a 进行初始化之前我们无法进行访问。其实我们可以想到函数表达式也同样会报这个错。
:::info
let , const 没有变量提升！！！
:::
